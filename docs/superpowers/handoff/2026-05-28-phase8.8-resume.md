### 남은 미해결 이슈

#### 이슈 1: "ㄱ 모양" 산발 마커 (미조사)

사용자가 특정 시트(시트 8 또는 9 추정 — 비교적 빈 영역)에서 이런 화면을 제보:
- 거의 빈 시트에 "ㄱ" (직각 모양) 마커들이 산발적으로 흩어져 있음
- 우측 하단에 `*NOTE` + `기준 선물통은 공사실...` 텍스트 일부 보임
- 건물 도면이 없고 레전드/노트 영역으로 추정

**원인 미확인** — 아래 중 하나일 가능성:
1. **LEADER arrowhead** — `DxfLeader.vertices.firstOrNull()` centroid가 화살표 끝점 → arrowhead 기호만 보임
2. **DIMENSION정의점 표시** — `DxfDimension.definitionPoint` centroid로 bbox는 계산되지만 anonymous block 안에 ㄱ 모양 SOLID가 있을 경우
3. **작은 INSERT 심볼** — num_owned=0~2 짜리 빈 블록인데 삽입점에 아무것도 안 그려짐 → 이 경우 버그 아님
4. **DxfUnknown 누락 엔티티** — 직각 기호 자체가 렌더 안 되는 엔티티 타입

> **Note:** Phase 8.7 이전에 비슷한 "ㄱ" 보고가 있었는데 그건 줌아웃 시 작은 가구/설비 심볼이 ㄱ처럼 보인 것 (버그 아님으로 결론). 이번은 **시트가 비교적 비어 있는데** 마커들이 떠 있는 상황 → 별도 조사 필요.

**조사 방법:**
```
시트 8이나 9 탭 선택 → ㄱ 마커 하나 줌인 → 스크린샷
adb logcat | grep CleanCAD 로 entity 타입 분포 확인
```
