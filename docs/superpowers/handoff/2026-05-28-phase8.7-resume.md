## Phase 8.7 작업 계획

### 목표
한국 건축 DWG의 model-space에 흩어진 시트들을 자동 클러스터링하고, 사용자가 시트 단위로 탐색할 수 있는 UI 제공.

### 제안 작업

#### Task A — 시트 클러스터링 알고리즘 (native 또는 Kotlin)

**입력:** 100K entities의 bounding box 리스트
**출력:** 클러스터 N개, 각 클러스터의 bounding box

옵션 1 (단순): **Grid 기반 클러스터링**
1. 전체 extents를 100×100 그리드로 나눔
2. 각 셀의 엔티티 개수 카운트
3. 셀 밀도 < threshold인 셀은 "빈 공간"으로 마크
4. 연결된 dense 셀들을 BFS/Union-Find로 cluster
5. 각 cluster의 union bbox = 시트 영역

장점: 단순, O(N), 메모리 적음
단점: 시트 간 거리가 가까우면 합쳐질 수 있음

옵션 2 (정밀): **DBSCAN on entity centroids**
1. 각 entity의 중심점 추출
2. DBSCAN (eps = ~5000mm, minPts = 30)
3. cluster 결과 = 시트

장점: 정밀
단점: 100K 포인트 DBSCAN은 O(N²) 또는 KD-Tree 필요

**추천:** 옵션 1로 시작. 한국 건축 도면 시트는 보통 수 미터 이상 떨어져 있으므로 grid 기반으로 충분.

**구현 위치:** Kotlin `Drawing` model에 `sheets: List<Sheet>` 필드 추가. `DrawingViewModel` parse 단계에서 클러스터링 수행.

```kotlin
data class Sheet(
    val id: Int,
    val name: String?,        // 시트 타이틀 블록의 텍스트에서 추출 (옵션)
    val bbox: BoundingBox,
    val entityIndices: IntArray  // 이 시트에 속한 엔티티 인덱스
)
```

#### Task B — 시트 선택 UI

옵션 1: **상단 탭바 (TabLayout)**
- 시트 N개를 탭으로 노출
- 탭 클릭 → fit-to-screen of bbox
- "전체 보기" 탭 추가

옵션 2: **하단 시트 드로어 (BottomSheet)**
- 시트 썸네일 + 이름 리스트
- 탭하면 해당 시트로 줌

**추천:** Task A 끝난 후 디자인 확정. 일단 Task A부터.

**구현 위치:** `ViewerActivity` + `activity_viewer.xml`. `DrawingView`에 `showSheet(sheetId: Int)` API 추가.

#### Task C — 시트 이름 자동 추출 (옵션, nice-to-have)

각 클러스터 내 TEXT/MTEXT 엔티티 중:
- `SHEET-` prefix 또는
- 큰 글자 (높이 > 1000mm) 또는
- 시트 타이틀 블록 영역 내 텍스트

를 후보로 이름 추출. 실패하면 "시트 1", "시트 2"...

#### Task D — 시트 외 영역 렌더링 옵션

선택된 시트만 보일 때, 다른 시트는:
- 옵션 A: 전혀 안 그림 (clipping)
- 옵션 B: 흐릿하게 (alpha 0.1) 보여서 컨텍스트 유지
- 추천: 옵션 A. 사용자가 "한 번에 한 시트"를 원함.

## 알려진 위험/주의사항

1. **`bit_TV_to_utf8` 메모리** — codepage가 CP_UTF8이면 src를 그대로 반환. `tv_to_utf8()`이 strdup으로 항상 malloc된 사본 반환하므로 호출자는 `free()` 안전. 깨면 LibreDWG 내부 메모리 손상 → crash.
2. **LibreDWG 필드명 deviation** — 플랜과 LibreDWG 0.13.4 다른 경우:
   - `dwg_getall_LAYER()` (not `dwg_get_layers()`)
   - `dwg->header.codepage` (not `from_version`) for `bit_TV_to_utf8`
   - `lay->frozen/off/locked` (not `lay->flag` bitmask)
   - `Dwg_HATCH_Path.num_segs_or_paths` (둘 다 케이스에 사용)
   - `MTEXT.x_axis_dir` 으로 rotation 계산 (no `.rotation` field)
   - `DIMENSION_COMMON.block` (BITCODE_H) → `block->obj->tio.object->tio.BLOCK_HEADER`
   - `DIMENSION_COMMON.clone_ins_pt/ins_scale/ins_rotation` (INSERT-style 변환 필드)
3. **MAX_ENTITIES = 100K** — `dwg_serialize.h`에 상수. 더 큰 도면이 잘리는 경우만 증가. ref.dwg는 이미 100K trunc 상태.
4. **Windows CRLF** — LibreDWG submodule의 `configure.ac` CRLF 변환은 `CMakeLists.txt`에서 strip. submodule 워킹트리가 dirty인 게 정상.
5. **emulator storage 부족** — `pm uninstall` + `pm trim-caches`로 정리. installDebug 반복 시 자주 발생.
6. **forward declaration 순서** — `write_dimension`이 `write_entity`를 호출하므로 forward declaration이 dwg_serialize.c 상단에 필요. 매크로(`DWGB_MAX_ENTITIES` 등)도 파일 최상단으로 이동했음. 새 entity expansion 함수 추가 시 동일 패턴 따를 것.
